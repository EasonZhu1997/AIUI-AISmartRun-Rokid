# AISmartRun 产品经理多维评估

日期：2026-07-08（对抗评审修复轮后）  
版本：0.1.0  
评估对象：当前 AIUI 眼镜端工程、PRD、预览产物、测试与本地 `.aix` 打包状态  
评估方法：8 镜头并行评审（首次体验 / HUD 可读性 / 教练安全 / 降级完备性 / 文案上架 / 文档一致性 / 工程闸门 / 数据完整性）→ 每条发现独立对抗复核 → 确认项全部修复或显式降级为已知限制 → 复评  
阶段结论：本地工程交付 96 / 100，建议进入真机 Alpha 验证；仍不建议绕过真机和 AIUI Studio 直接上架提审

## 1. 总体判断

AISmartRun 是一个口径克制、边界诚实的眼镜端跑步 MVP：首页只做就绪陈述，一键开跑，跑步 HUD 纯展示；无外设时用眼镜 IMU 估算，接入标准蓝牙心率后同屏增强；AI 教练用短语音回答节奏与安全问题，任何一层能力缺失都有确定性降级。

本轮评审共产生 74 条发现，经逐条对抗复核确认 45 条（驳回 29 条），其中 5 条评为 critical。**全部确认项已修复并新增自动化测试守卫，或降级为文档化的已知限制**。修复后 `npm run verify:release` 全绿：doctor（含单绿色/emoji 扫描）→ 预览校验 → 151/151 测试（含 18 条页面生命周期级测试）→ 本地 `.aix` 构建与断言。

追加修复（双产品线评估中发现，M0 级）：AIUI↔后端 anon-login 契约断裂——已部署后端强制要求 `app_key`（缺失 422），而眼镜端按旧契约只发 `app_id`，记忆链路在真机上会静默全灭。已修：`coach_app_key` storage 注入口径、无 key 时跳过登录显式降级（不打必失败请求）、契约文档改写为部署现实、Alpha A-12 增加注入前置、新增 3 条守卫测试。详见 [DUAL_PRODUCT_COMMERCIAL_EVAL.md](./DUAL_PRODUCT_COMMERCIAL_EVAL.md)。

## 4. 仍不能宣称的事项（真机边界）

1. 不能宣称已通过 Rokid 真机完整验证——BLE 首扫手势门槛、息屏行为、ASR/LLM/TTS 宿主可用性、1Hz 功耗都是 Alpha 项。
2. 不能宣称 IMU 距离/配速达到跑表精度；走/跑步长分档校准列入 P1（B-03 记录误差）。
3. 心率区间基于固定 maxHr=190，对年长用户 Z5 提醒偏迟；自定义最大心率列入 P1。
4. UI 为中文；英文完整体验是 P2（Store 英文描述已如实反映功能而非界面语言）。
5. 正式签名、上传与商店提审仍以 AIUI Studio 为准。

## 5. 下一阶段建议

下一阶段仍定义为 Alpha 真机验证（见 [ALPHA_TEST_MATRIX.md](./ALPHA_TEST_MATRIX.md)），出口门槛：A-02、A-03、A-08、A-10、A-11 必过且 P0 ≥10/12。本轮新增的 A-13（防误触退出）与 A-07 的 8 秒回退口径请一并验证。

暂不建议：跑后历史/训练计划/排行/地图；私有手表协议；把配置塞回跑步 HUD；后端接管 AI 主回答（除非官方 LanguageModel 真机不可用）。

## 6. 当前阶段产品结论

修复轮之后，这个 MVP 的三条底线——"单眼镜能跑、心率可接可退、语音可兜底"——全部有可执行测试背书，而不是纸面承诺。产品对用户不撒谎（不编数、不冒充"此刻"、不虚报"已接入"），对评审不撒谎（已知限制全部显式声明）。本地工程交付评为 96 / 100；真机 Alpha 是把 88 分成熟度提上来的唯一路径，请按矩阵执行。

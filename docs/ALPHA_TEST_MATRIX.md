# AISmartRun Alpha 真机测试矩阵

日期：2026-07-08  
阶段：Alpha 准入测试  
目标：验证单眼镜可跑、心率可接、语音可兜底、资源可恢复。

## 测试前准备

| 项 | 要求 |
|---|---|
| 眼镜 | Rokid Glasses，已开启 AIUI 调试。 |
| 手机 | Rokid AI APP 已登录，可运行当前 Agent。 |
| 包 | 使用 `npm run build:local` 生成的 `release/AISmartRun-current.aix` 做本地检查；正式真机包走 AIUI Studio。 |
| 心率设备 | Garmin Fenix 8 广播心率，或 ESP32 HRS 模拟器。 |
| 网络 | 手机侧网络可用；断网场景单独测试。 |

## P0 测试用例

| ID | 场景 | 步骤 | 通过标准 |
|---|---|---|---|
| A-01 | 首屏进入 | 打开 AISmartRun | 默认进入首页；显示单眼镜已就绪和心率就绪状态（已关闭 / 不可用 / 已记住 / 待连接之一）；首页不发起蓝牙连接；无错误弹窗。 |
| A-02 | 一键开跑 | 首页触发开跑 | 3 秒内进入 HUD 并开始计时。 |
| A-03 | 无心率跑步 | 不开任何心率设备，跑/走 10 分钟 | HUD 不显示心率列；时长持续刷新；步频有动作响应；不崩溃。 |
| A-04 | 加速度计不可用兜底 | 在浏览器 Craft 预览（宿主未注入 Accelerometer）打开 run_hud | 显示仅计时态，不崩溃。 |
| A-05 | 设备页授权 | 进入设备页，扫描 Fenix 8 / ESP32 HRS | 可发现标准心率设备，并保存为首选设备。 |
| A-06 | 已记住设备优先连接 | 保存设备后回首页开跑进入 HUD | HUD 优先尝试已记住设备；连接成功后显示有效 bpm。 |
| A-07 | 心率断连 | 跑步中关闭心率广播 | HUD 静默回到单眼镜模式，跑步不中断；停止广播后 8 秒内回退单眼镜。 |
| A-08 | 息屏自动暂停与恢复 | 跑步中息屏 1 分钟，然后回到 HUD | 暂停期间时长不增加；恢复后 10 秒内步频恢复刷新；无双倍跳秒。 |
| A-09 | AI 语音提问 | 跑步中进入教练页，按键/点击开始说话 | ASR 成功识别；LanguageModel 返回短答；TTS 播放或文本可读。 |
| A-10 | AI 兜底 | 手机开飞行模式或断开代理网络后提问 | 本地规则回答，不编造心率/距离/配速。 |
| A-11 | 物理 Backspace | 在首页、设备、设置、HUD、教练页分别按 Backspace | 首页退出；设备/设置返回；HUD 双击确认后释放资源返回（见 A-13）；教练进行中取消，否则返回。 |
| A-12 | EverMind 后端记忆链路 | **前置：先在眼镜 wx storage 注入 `coach_app_key`（后端 anon-login 必填共享 key，见 BACKEND_EVERMIND_CONTRACT.md）**，再完成一次 AI 教练问答 | 匿名登录成功换到 JWT；回答中出现记忆上下文，或后端日志有本轮记录；未注入 key 时客户端跳过登录、记忆显式关闭但回答不受影响；记忆/记录失败时也必须有回答。 |
| A-13 | 防误触退出 | 跑步中单击一次 Backspace，等 3 秒；再测 3 秒内连按两次 | 单击后显示“再按一次结束”，3 秒内不按第二次则继续跑步不退出；3 秒内双击才结束并返回。 |
| A-14 | 跑步记录落库 | 注入 `coach_app_key` 后跑 2 分钟以上，双击退出回到首页，等待数秒 | 后端 `GET /api/runs` 可见一条 `source="aiui"` 记录（时长/距离/均配一致）；断网退出时记录保留在待传队列，恢复网络后再进首页可补传成功。 |

## P1 观察项

| ID | 观察项 | 记录 |
|---|---|---|
| B-01 | HUD 开跑自动连接是否被交互态拦截 | 记录是否需要先进入设备页授权。 |
| B-02 | 1Hz 刷新功耗和发热 | 连续 20 分钟主观观察。 |
| B-03 | IMU 距离误差 | 400m 或 1km 对比跑表；记录误差百分比。 |
| B-04 | TTS 延迟 | 从回答生成到播报开始的延迟。 |
| B-05 | 英文 UI 宽度 | 英文预览和真机是否溢出。 |
| B-06 | AIUI Studio 包差异 | 记录正式包与本地 `.aix` reader 检查结果是否一致。 |
| B-07 | CXR-L 共存 | 同一副眼镜：手机装 side-load 版 relay-apk（CXR-L HUD）+ 眼镜装 AIUI Agent，观察两者能否共存/切换、是否互相抢占显示；结论回写 [DUAL_PRODUCT_COMMERCIAL_EVAL.md](./DUAL_PRODUCT_COMMERCIAL_EVAL.md) 缝 3。 |

## 退出 Alpha 的最低门槛

与 `docs/LOCAL_RELEASE_SCORECARD.md` 使用同一套门槛：

1. 必过项：A-02、A-03、A-08、A-10、A-11 全部通过。
2. P0 用例（A-01 至 A-12）至少 10/12 通过。
3. 若 A-06 不通过，必须把产品口径改为“手动授权后自动优先连接”。
4. 若 A-09 不通过，但 A-10 通过，可保留为文本教练 Alpha；上架前必须补语音闭环。
